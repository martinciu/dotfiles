House rules (per CLAUDE.md)

  • Hand-rolled status bar. No catppuccin/powerline plugins.
  • Prefix is C-a, not C-Space.
  • No bind -n M-* — Alt is reserved for Polish diacritics.
  • Windows auto-rename from the active pane's last typed command (zsh preexec sets @last_cmd). allow-rename off stays so OSC titles can't override.
  • Worktree status segment uses git rev-parse --git-dir vs --git-common-dir. Don't replace with git worktree list parsing.

From the shell

tmux-resurrect + tmux-continuum auto-save sessions every 15 min and auto-restore on a fresh server. See the "Persistence" card below for bindings.

Sesh picker prefix t

C-at opens a popup running sesh picker -i -d -H — full vanilla, all sources (configured / tmux / zoxide / tmuxinator). The shared config lives at .config/sesh/sesh.toml with an import directive pulling machine-local sessions from ~/.config/sesh/sesh.local.toml (untracked, copied from a template by bootstrap.sh).

_ZO_EXCLUDE_DIRS blocks ~/, ~/Downloads/*, ~/.config/*, ~/Library/* from zoxide indexing — keeps the picker focused on real projects under $PROJECTS_HOME. Don't re-introduce a custom fzf wrapper script.

Persistence resurrect + continuum

C-a C-ssave snapshot now (manual)
C-a C-rrestore latest snapshot (manual)
auto-saveevery 15 min (continuum default)
auto-restoreon fresh tmux server start (@continuum-restore 'on')
snapshots in~/.local/share/tmux/resurrect/

Defaults only: no @resurrect-processes (panes get fresh shells in cwd; nvim/ssh are NOT relaunched), no @resurrect-capture-pane-contents (scrollback is not saved). Auto-restore only fires on a brand-new server with zero sessions, so attaching to an already-running server (sesh's normal flow) is unaffected.

Auto-rename (custom)

Window name follows the active pane's last typed command, set by zsh preexec via @last_cmd. Env-var assignments are stripped first; the first two whitespace-separated tokens are used.

  • vim foo.rb → window name: vim foo.rb
  • FOO=bar bundle exec rspec → bundle exec
  • Idle pane / non-zsh shell → falls back to pane_current_command (usually zsh)

allow-rename off stays — OSC titles from apps (e.g. Claude Code) cannot override.

Status bar / hand-rolled Solarized

Layout

Bottom bar, bg=base02 / fg=base0. Three segments:

  • Left: session name on a blue chip with a powerline arrow.
  • Center: windows. Inactive = base01 on base02; active = yellow-on-base02 chip.
  • Right: project name (violet chip) → git/worktree status from tmux-git-status.

Solarized palette in use

base03 base02 base01 base0 base2 base3
blue (session) yellow (active win) violet (project) cyan (selection) red

Selection / message style uses cyan-on-base03; pane border switches base01 → blue when active.

Config notes / ~/.config/tmux/tmux.conf

# Prefix
set -g prefix C-a            # C-Space conflicts with macOS input-source switching

# Terminal — truecolor
set -as terminal-features ",xterm-ghostty:RGB"
set -as terminal-features ",xterm-256color:RGB"

# Sane defaults
set -g mouse on
set -g base-index 1
set -g renumber-windows on
set -g history-limit 50000
set -g escape-time 10          # vim-friendly
set -g focus-events on

# Bells silenced everywhere
set -g bell-action  none
set -g visual-bell  off
set -g monitor-bell off

# Window naming follows active pane via @last_cmd (set in zsh preexec)
set -g allow-rename off
set -g automatic-rename on
set -g automatic-rename-format '#{?#{==:#{@last_cmd},},#{pane_current_command},#{@last_cmd}}'
