Languages enabled

LazyVim "extras" loaded — full LSP / formatter / linter setup:

  • Ruby (ruby-lsp, rubocop, erb-formatter/lint)
  • TypeScript / JavaScript (vtsls, eslint-lsp)
  • Tailwind CSS (tailwindcss-language-server)
  • JSON, YAML, Markdown (+ marksman, markdownlint, markdown-toc)
  • Lua (lua-language-server, stylua)
  • Shell formatting (shfmt)

Treesitter highlighting is also on for Python and SQL (no LSP).

First-run checklist

  1. Open nvim: nvim
  2. Lazy will install plugins on first start; wait for the splash to settle.
  3. Run :checkhealth — fix any red items (clipboard, node, ripgrep).
  4. Run :Mason — confirm everything from mason-lock.json is installed.
  5. Open a file in a project to trigger LSP attach; check :LspInfo.

House rules (per CLAUDE.md)

  • No Alt mappings. Alt is reserved for Polish diacritics. <A-j>/<A-k> are explicitly removed.
  • Solarized + JetBrainsMono Nerd Font only.
  • Lockfiles committed: lazy-lock.json, mason-lock.json.
  • Mason: :MasonLock snapshots, :MasonLockUpdate upgrades + snapshots.
  • .claude/worktrees/ is hidden from pickers and built-in completion. Snacks excludes it from files, grep, explorer, and recent (see lua/plugins/snacks.lua); wildignore in lua/config/options.lua drops it from :e/glob expansion. If ␣␣ or ␣/ can't find a file under there, that's why.

Tip

For project-wide grep, prefer ␣/ over ␣sg — both are live grep, but the first is the "fast lane" alias.

Start typing inside a picker; results filter as you type. Use ! at the start to negate a term. Glob-ish path filters work in file pickers ('src, !test).

Auto-server

When launched inside tmux, nvim auto-listens on $XDG_RUNTIME_DIR/nvim-tmux-<session>.sock (Linux) or $TMPDIR/nvim.$USER/nvim-tmux-<session>.sock (macOS) so the <prefix> o tmux binding can remote-send :edit.

Removed defaults

  • <A-j>, <A-k> (move-line) — Alt is reserved for Polish diacritics.
  • gzip, tarPlugin, tohtml, tutor, zipPlugin are disabled in lazy.lua.
